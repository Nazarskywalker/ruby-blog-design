# Ruby Blog

Простий блог на Ruby з використанням фреймворку Sinatra.

## Функціональність

- ✅ Створення нових постів
- ✅ Перегляд списку всіх постів
- ✅ Перехід на окрему сторінку поста
- ✅ Редагування існуючих постів
- ✅ Видалення постів
- ✅ Responsive дизайн

## Вимоги

- Ruby 2.7+
- Bundler

## Встановлення та запуск

1. Встановіть залежності:
```bash
bundle install
```

2. Запустіть сервер (рекомендований спосіб):
```bash
ruby start_server.rb
```

Альтернативні способи запуску:
```bash
# Через rackup (більш стабільний)
rackup config.ru -p 4567 -o 0.0.0.0

# Класичний спосіб
ruby app.rb
```

**Якщо сервер зупиняється:** використовуйте `rackup config.ru -p 4567 -o 0.0.0.0`

3. Відкрийте браузер за адресою: http://localhost:4567

## Доступ з інших пристроїв

Для доступу з телефону чи інших пристроїв у тій же мережі Wi-Fi:

1. Запустіть сервер через `ruby start_server.rb`
2. Система покаже вашу IP-адресу
3. На телефоні відкрийте: `http://[ваша_IP]:4567`

**Приклад:** `http://192.168.1.100:4567`

## Оновлення в реальному часі

Сайт автоматично оновлюється при змінах:
- Вимкнено кешування браузера
- Зміни відображаються миттєво на всіх пристроях
- Пости зберігаються в пам'яті сервера

## Структура проєкту

```
ruby-blog/
├── app.rb              # Основний файл додатку
├── Gemfile             # Залежності
├── views/              # HTML шаблони
│   ├── layout.erb      # Основний layout
│   ├── index.erb       # Головна сторінка
│   ├── new_post.erb    # Форма створення/редагування
│   └── show_post.erb   # Сторінка поста
├── public/             # Статичні файли
│   └── style.css       # Стилі
└── README.md           # Документація
```

## Використання

1. **Головна сторінка** - показує всі пости
2. **Створити пост** - кнопка "Створити пост" у верхньому меню
3. **Перегляд поста** - клікніть на заголовок поста
4. **Редагування** - кнопка "Редагувати" на сторінці поста
5. **Видалення** - кнопка "Видалити" з підтвердженням

## Примітки

- Пости зберігаються в пам'яті (при перезапуску сервера дані зникнуть)
- Для постійного зберігання можна додати базу даних (SQLite, PostgreSQL)
